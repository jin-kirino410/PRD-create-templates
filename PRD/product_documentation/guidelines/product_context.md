# 事業/プロダクト前提ガイドライン

## 概要
このガイドラインは、PRD作成前に確認すべき事業・プロダクトの前提事項を整理したものです。これらの前提を明確にすることで、より適切なPRDを作成できます。

## 1. 事業戦略の前提

### 1.1 事業ビジョン
- [ ] 会社の長期的なビジョンは何か？
- [ ] このプロダクトは事業ビジョンのどの部分を支えるか？

### 1.2 事業目標
- [ ] 今年度の事業目標は何か？
- [ ] このプロダクトはどの事業目標に貢献するか？

### 1.3 競合状況
- [ ] 主要な競合他社はどこか？
- [ ] 競合他社の強み・弱みは何か？
- [ ] 差別化ポイントは何か？

## 2. 市場環境の前提

### 2.1 市場規模
- [ ] ターゲット市場の規模はどの程度か？
- [ ] 市場の成長性はどうか？

### 2.2 市場トレンド
- [ ] 業界の主要なトレンドは何か？
- [ ] 技術的な変化はあるか？
- [ ] 規制の変更は予想されるか？

### 2.3 顧客ニーズ
- [ ] 顧客の主要な課題は何か？
- [ ] 顧客のニーズは変化しているか？

## 3. 技術的前提

### 3.1 技術戦略
- [ ] 会社の技術戦略は何か？
- [ ] 使用する技術スタックの方針は？
- [ ] 技術的負債の状況は？

### 3.2 技術的制約
- [ ] 既存システムとの統合要件は？
- [ ] セキュリティ要件は？
- [ ] パフォーマンス要件は？

### 3.3 技術的リスク
- [ ] 新技術の採用リスクは？
- [ ] 技術的な課題は予想されるか？

## 4. 組織的前提

### 4.1 チーム構成
- [ ] 開発チームの規模・スキルは？
- [ ] 必要な外部リソースは？
- [ ] チームの経験レベルは？

### 4.2 予算・リソース
- [ ] プロジェクト予算は？
- [ ] 人的リソースの制約は？
- [ ] 外部委託の可能性は？

### 4.3 スケジュール
- [ ] 絶対に守るべき期限は？
- [ ] 重要なマイルストーンは？
- [ ] 季節的要因はあるか？

## 5. 顧客・ユーザーの前提

### 5.1 ターゲットユーザー
- [ ] 主要なユーザー層は？
- [ ] ユーザーの技術レベルは？
- [ ] ユーザーの利用環境は？

### 5.2 ユーザー体験
- [ ] 既存のユーザー体験の課題は？
- [ ] ユーザーの期待値は？
- [ ] ユーザーの行動パターンは？

### 5.3 顧客関係
- [ ] 既存顧客との関係は？
- [ ] 顧客の声はどのように収集しているか？
- [ ] 顧客サポートの体制は？

## 6. 法的・規制的前提

### 6.1 法的要件
- [ ] 関連する法律・規制は？
- [ ] コンプライアンス要件は？
- [ ] データ保護要件は？

### 6.2 業界規制
- [ ] 業界特有の規制は？
- [ ] 認証・認可要件は？
- [ ] 品質基準は？

## 7. 前提事項の確認方法

### 7.1 情報収集
- [ ] 社内の関係者へのヒアリング
- [ ] 市場調査・競合分析
- [ ] 顧客インタビュー・調査
- [ ] 技術的検証・検討

### 7.2 前提の文書化
- [ ] 前提事項の明文化
- [ ] 前提の根拠・出典の記載
- [ ] 前提の変更可能性の評価
- [ ] 前提の共有・承認

### 7.3 前提の検証
- [ ] 前提の妥当性確認
- [ ] 前提の変更リスク評価
- [ ] 前提変更時の影響分析

## 8. 前提事項の活用

### 8.1 PRDへの反映
- [ ] 前提事項をPRDの冒頭に記載
- [ ] 前提に基づく制約事項の明記
- [ ] 前提変更時の対応方針の記載

### 8.2 前提の管理
- [ ] 前提事項の定期的な見直し
- [ ] 前提変更時の関係者への通知
- [ ] 前提変更時のPRD更新
